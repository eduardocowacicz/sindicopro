# Projeto Revisado

Esta pasta concentra a documentação principal do sistema de gestão condominial. Ela funciona como a base oficial para entender o escopo do MVP, as regras do negócio e a orientação usada na criação dos mockups.

## Arquivos desta pasta

| Arquivo | Finalidade |
| --- | --- |
| [ResumoDeFuncoesRevisado.txt](ResumoDeFuncoesRevisado.txt) | Visão executiva do projeto, com objetivo, prioridades do MVP e funções principais |
| [LevantamentoDeRequisitosRevisado.txt](LevantamentoDeRequisitosRevisado.txt) | Requisitos funcionais, regras de negócio, requisitos não funcionais e itens fora do escopo |
| [PromptPlanejamentoDesenvolvimento.txt](PromptPlanejamentoDesenvolvimento.txt) | Direcionamento usado para planejar a experiência e as telas dos mockups |

## Ordem recomendada de leitura

1. Comece pelo [ResumoDeFuncoesRevisado.txt](ResumoDeFuncoesRevisado.txt)
2. Depois leia o [LevantamentoDeRequisitosRevisado.txt](LevantamentoDeRequisitosRevisado.txt)
3. Por fim, consulte o [PromptPlanejamentoDesenvolvimento.txt](PromptPlanejamentoDesenvolvimento.txt) para entender a lógica de UX e navegação

## O que está consolidado aqui

- objetivo do sistema
- contexto do condomínio
- prioridade do MVP
- módulos principais
- regras de acesso e histórico
- regras financeiras e de fechamento
- diretrizes de usabilidade, segurança e auditoria
